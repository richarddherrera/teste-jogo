# Arena Nexus UI - Especificação de Frontend

## Visão Geral

O Arena Nexus UI é uma interface de alta performance projetada para operar em sinergia com o backend Java. O conceito central é transformar a experiência do usuário de uma simples navegação para um **"Dashboard de Comando"** (Command Center), onde cada elemento visual serve a um propósito funcional de telemetria ou tomada de decisão.

### Filosofia de Design: Pro-Circuit Aesthetic

A interface deve respirar competitividade, utilizando princípios de design que priorizam:
- Clareza sob pressão
- Densidade de dados sem sobrecarga cognitiva
- Imersão sensorial que reflete a eletricidade dos grandes torneios

Glassmorphism e Superfícies de ComandoO uso de superfícies em vidro (glassmorphism) permite a sobreposição de camadas de informação sem perder o contexto do que está por baixo. Este efeito é alcançado através da combinação de transparências, desfoque de fundo (backdrop-blur) e bordas sutis que simulam a refração da luz em superfícies sintéticas. No Arena Nexus, o glassmorphism é elevado ao conceito de "Liquid Glass", onde animações suaves de gradientes e distorções criam uma interface que parece viva e tecnológica, evocando a estética de monitores militares de alta tecnologia ou cockpits de simulação.A aplicação de bordas afiadas é uma escolha deliberada para romper com a tendência de "suavização" (rounded corners) das interfaces móveis tradicionais. Cantos retos e chanfrados comunicam precisão e rigor técnico, afastando o Arena Nexus da aparência de uma rede social e aproximando-o de uma ferramenta de engenharia competitiva.2. O Dashboard do Jogador (The Profile): Visualização de Performance e ELOO coração da experiência do usuário no Arena Nexus é o Perfil do Jogador, que atua como um registro biométrico de sua carreira competitiva. O foco absoluto nesta tela é a visualização do ELO e da progressão.2.1. O Card de Rank DinâmicoO Card de Rank não é apenas um exibidor de ícones; ele é um componente reativo que altera sua morfologia visual com base nos dados retornados pelo back-end Java. Se o sistema identifica que o jogador atingiu a categoria DIAMANTE, o front-end deve disparar um estado visual que altera a propriedade box-shadow para um brilho ciano neon e aplica um filtro de reflexo metálico dinâmico na superfície do card.A implementação técnica deste dinamismo utiliza o sistema de temas do Tailwind CSS combinado com Framer Motion. Através de variantes de estado, o componente transiciona suavemente entre os estilos de cada categoria, garantindo que a subida de nível seja sentida pelo usuário como um evento de grande impacto.2.2. Gráficos de Telemetria CompetitivaPara honrar a lógica de processamento de dados do Java, o front-end utiliza o Recharts para plotar o histórico de ELO das últimas 10 partidas. Diferente de gráficos financeiros tradicionais, o gráfico do Arena Nexus deve utilizar linhas neon com brilho externo (glow effects) e áreas preenchidas com gradientes semitransparentes.O cálculo da inclinação da curva de progresso permite ao sistema fornecer insights preditivos. Se a taxa de ganho de ELO for superior a um determinado coeficiente, o sistema pode exibir uma tag de "On Fire", incentivando o jogador a continuar sua jornada no matchmaking.2.3. O Medidor de Integridade (Barra de Penalidade)A gestão da "Toxicidade" ou "Status de Alerta" é crucial para manter o ecossistema saudável. O Arena Nexus implementa uma barra de penalidade que utiliza a lógica de animação pulsante para níveis críticos. Quando o back-end reporta duas ou mais penalidades (RN06), a barra transiciona de um estado neutro para um vermelho escuro pulsante (animate-pulse), sinalizando que o próximo erro resultará em um banimento automático.Esta visualização de risco é baseada no princípio de feedback preventivo, onde a interface atua como um conselheiro para o jogador, em vez de apenas um executor de punições. A pulsação da barra cria uma resposta psicológica de urgência, essencial para a moderação de comportamento em ambientes de alta pressão.3. O Ecossistema de Matchmaking (The Lobby): Feedback Visual e Lógica FIFOO matchmaking é o ponto onde a teoria do back-end (filas FIFO) encontra a realidade da experiência do usuário. No Arena Nexus, o lobby é tratado como uma "fase de pré-combate", onde o design deve mitigar a ansiedade da espera e preparar o jogador para a partida.3.1. A Fenomenologia da Espera: Animação de ScanningAo acionar o botão "Find Match", o Arena Nexus inicia uma sequência de animação de "Scanning". Utilizando gradientes lineares animados e filtros de desfoque, o front-end simula uma varredura de radar sobre a base de dados global. Esta técnica, conhecida como "perceived performance optimization", transforma o tempo de espera em um momento de antecipação.Tecnicamente, o cronômetro é sincronizado com o servidor via WebSockets para garantir precisão absoluta. Enquanto a fila é processada, a interface exibe o "Filtro de Categoria Visual", informando ao jogador quais faixas de ELO estão sendo analisadas (ex: "Buscando jogadores Ouro ou Prata"). Esta transparência visual valida a lógica do algoritmo e reduz a percepção de injustiça no pareamento.3.2. Notificação de Match Found e Aceite de PartidaQuando a fila FIFO identifica um par compatível, o sistema dispara um pop-up de alta prioridade. Este componente utiliza AnimatePresence do Framer Motion para entrar na tela com um efeito de escala e desfoque reverso. O som de alerta deve ser sincronizado com um brilho neon periférico na tela, garantindo que o jogador, mesmo que esteja em outra aba, perceba que a partida foi encontrada.O botão de aceite utiliza um medidor de progresso circular que se esgota em 10 segundos. Se o jogador não confirmar a presença, o sistema registra automaticamente uma penalidade por não comparecimento, atualizando a barra de toxicidade no perfil.4. Gestão de Torneios (The War Room): Arquitetura de Chaves e RankingsA visualização de torneios no Arena Nexus, denominada "The War Room", é projetada para fornecer uma visão macroscópica da competição. Aqui, a complexidade dos dados de Streams do Java é traduzida em estruturas geométricas limpas.4.1. Visualização em Árvore de Chaves (Brackets)Para renderizar as chaves de eliminação, o front-end utiliza bibliotecas especializadas como @g-loot/react-tournament-brackets, que permitem a customização completa dos nós de partida através de styled-components. As linhas de conexão entre as partidas são tratadas como condutores de energia visual: se uma partida está EM_ANDAMENTO, a linha entre os nós brilha intensamente, indicando o fluxo ativo da competição.Esta visualização é fundamental para que espectadores e jogadores entendam a progressão do torneio em tempo real. Cada nó de partida exibe estatísticas rápidas via hover, permitindo comparar o Win Rate acumulado dos competidores antes mesmo do início da partida.4.2. Leaderboard de Alta Performance e Streams de DadosA Tabela de Classificação do Arena Nexus é uma vitrine da elite competitiva. Através do consumo de Streams de dados, a tabela exibe estatísticas complexas como "KDA médio" e "Sinergia de Time" sem perda de performance de renderização. O Top 3 é destacado com medalhas dinâmicas que utilizam reflexos metálicos baseados no movimento do mouse (gyroscope-like effects).A implementação utiliza virtualização de listas (react-window) para garantir que rankings com milhares de jogadores possam ser navegados com fluidez total, mantendo o efeito de glassmorphism em cada linha da tabela.5. Especificação Técnica e Stack Tecnológica: Arena Nexus UIPara materializar esta visão, a stack tecnológica foi selecionada com base em três pilares: velocidade de desenvolvimento, fidelidade visual e reatividade de dados.5.1. O Núcleo do Front-End: React e Next.jsA escolha do Next.js como framework principal justifica-se pela sua excelência no gerenciamento de estados de listas e roteamento otimizado. Para uma aplicação que lida com rankings globais, a capacidade de realizar Server-Side Rendering (SSR) para SEO e Static Site Generation (SSG) para performance de carregamento é vital.A gestão de estado global, especialmente para os dados de matchmaking e torneios, é realizada através do TanStack Query (React Query). Esta biblioteca permite sincronizar o estado da interface com a lógica Java do back-end de forma transparente, lidando com cache, refetching automático e estados de carregamento de forma elegante.5.2. Estilização e Design System: Tailwind CSS v4O Tailwind CSS é a ferramenta definitiva para o estilo "Pro-Circuit". A sua natureza utility-first permite criar os efeitos complexos de glassmorphism e neon diretamente nas classes HTML, reduzindo drasticamente o volume de arquivos CSS personalizados.A configuração do tailwind.config.js no Arena Nexus inclui uma paleta de cores customizada com variáveis CSS para os tons neon e metálicos, permitindo que o sistema de cores seja alterado dinamicamente caso o jogador mude de Rank ou o torneio tenha uma identidade visual própria.5.3. Visualização de Dados e ÍconesGráficos: Recharts é utilizado pela sua natureza declarativa e integração perfeita com componentes React, permitindo que os gráficos de ELO sejam responsivos e animados por padrão.Ícones: Lucide React fornece uma vasta gama de ícones técnicos e minimalistas. Para elementos específicos de combate, o FontAwesome Pro é integrado para garantir acesso a escudos, espadas e troféus com alta fidelidade visual.6. Este componente utiliza um gráfico de radar para plotar a média de ELO de cada membro do time em diferentes eixos (Mecânica, Estratégia, Comunicação, Consistência). Experiência de Usuário: Os "Mimos" de UX para o Pro-CircuitO diferencial competitivo de um produto digital reside na atenção aos detalhes micro-interativos. No Arena Nexus, esses detalhes são chamados de "Mimos de UX" e visam recompensar a performance do jogador e reforçar a seriedade da moderação.7.1. Transição de Tier e Celebração de ConquistaSubir de OURO para DIAMANTE é um rito de passagem. Notificações de WO e Penalidades EstilizadasQuando uma partida é finalizada por WO (W.O.), o sistema não exibe apenas um texto simples. Ele dispara um alerta estilizado: "⚠️ Jogador X penalizado por não comparecimento." Este alerta utiliza um fundo vermelho escuro com uma borda neon pulsante, garantindo que a consequência da infração seja visualmente pesada e memorável.8. Considerações de Performance e Real-Time no Front-EndPara manter o "Dashboard de Comando" responsivo, o Arena Nexus adota estratégias rigorosas de otimização de renderização e comunicação com o servidor.8.1. WebSockets vs Server-Sent Events (SSE)Para a comunicação em tempo real, o Arena Nexus utiliza uma abordagem dual. Enquanto as atualizações de chat e confirmações de matchmaking utilizam WebSockets (pela necessidade de comunicação bidirecional de baixa latência), os feeds de resultados de partidas e atualizações de ranking na "War Room" utilizam SSE. O SSE é mais eficiente para transmissões unidirecionais do servidor para o cliente, permitindo manter milhares de conexões abertas com menor custo computacional.Caso de UsoProtocoloBenefício para o JogadorAceite de PartidaWebSocketLatência sub-100ms para garantir o slot.Chat de TimeWebSocketDiálogo fluido sem atrasos na estratégia.Atualização de RankingSSEInformação sempre fresca sem necessidade de refresh.Status do TorneioSSEVisualização das chaves atualizada automaticamente.8.2. A Regra dos 3 Segundos e a Hierarquia VisualO design do dashboard segue a "Regra dos 3 Segundos": o usuário deve ser capaz de compreender seu status principal (Rank, ELO e próxima partida) em menos de três segundos após o carregamento da página. Para isso, utilizamos a hierarquia visual baseada no padrão de leitura em "F" (F-Pattern), posicionando os indicadores de ELO no canto superior esquerdo e o botão de matchmaking em uma posição central de destaque.9. Conclusão: O Futuro da Interface Arena NexusA construção da Arena Nexus UI não é apenas um exercício de desenvolvimento de front-end, mas a materialização de uma filosofia competitiva. Ao unir a robustez das lógicas Java (Streams, FIFO, Validações) com uma interface que domina o "Pro-Circuit Aesthetic", criamos uma ferramenta que respeita o tempo do jogador e a autoridade do administrador.A stack tecnológica proposta (Next.js, Tailwind v4, Framer Motion) garante que a aplicação seja escalável e capaz de suportar as demandas visuais de 2026. A integração de elementos como o Visualizador de Sinergia e as transições dinâmicas de Tier transforma números abstratos em experiências emocionais gratificantes. O Arena Nexus UI define o novo padrão para dashboards competitivos, onde cada pixel serve à glória do vencedor e à ordem do ecossistema.
